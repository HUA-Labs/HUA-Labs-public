# i18n 패키지 피드백 및 개선 방향 (2025-12-02)

## 📋 피드백 요약

### ✅ 장점

1. **명확한 문제 해결**
   - i18n에서 언어 변경 시 깜빡임 해결 (기존 i18next에서도 UX 문제였음)
   - 초기 SSR과 클라이언트 언어 mismatch → 하이드레이션 문제 해결

2. **적절한 추상화 레벨**
   - core / loaders / zustand adapter로 역할 분리
   - 필요에 따라 loader/adapter를 교체하거나 확장 가능

3. **Zustand 통합의 타입 안전성**
   - 기존 상태관리 통합 i18n 솔루션 중 타입 친화적인 구현

4. **비침투적 구조**
   - `createCoreI18n` → Provider로 감싸기만 하면 됨
   - 외부 프로젝트에 쉽게 이식 가능

5. **트리쉐이킹 고려 + SSR 호환성**
   - `sideEffects: false`, `initialTranslations`, `useRef` 루프 방지 등
   - 프로덕션 환경에서 필요한 디테일 반영

### ❌ 약점 / 보완점

1. **i18n 에코시스템 내 위치가 모호함**
   - 기존 i18next, next-intl, react-i18next, lingui, tolgee 등과의 차별점이 추상적
   - README에서 **"왜 다른 대안보다 더 낫다"**를 직접적으로 설명 필요

2. **캐싱 전략 고도화는 아직 보통 수준**
   - TTL 캐시, 전역 캐시, 중복 요청 방지는 기본
   - L2 캐시 (IndexedDB 등)나 사용자 우선언어 기반 프리로딩 전략 없음

3. **실제 번역 키 정적 분석/타입추론 기능 없음**
   - README에 "TS 완전 지원"이라 했지만
   - `t('key')` 호출 시 키 자동완성은 없음
   - 경쟁 프레임워크들이 적극 밀고 있는 기능

4. **국제화 복잡도 대응 범위가 한정적**
   - 복수형(pluralization)
   - 날짜/시간 형식화(intl)
   - Right-to-Left(RTL)
   - 메시지 fallback 체인
   - 상위 패키지에서 다룰 계획이라면 명시 필요

5. **릴리즈 전략, 예제 프로젝트 없음**
   - 기술은 좋지만 포지셔닝, 배포 전략, 커뮤니티 관점은 0단계
   - 예: next-i18n-example, zustand-i18n-starter, i18n-ssr-demo 같은 템플릿 리포 필요

### 🧭 외부 인상

**"실력 있는 인디 개발자가 만든, 현실적인 UX 문제를 해결하는 i18n 툴"**

하지만 동시에

**"아직 브랜드도 없고, 커뮤니티도 없고, i18next와 뭐가 다르냐는 질문에 한 줄로 답하기 어렵다"**

### 💬 한 줄 요약

**"기술적으로 잘 만든 건 분명한데, OSS로 확장하려면 메시지 정리와 실사용 설득이 더 필요하다."**

## 📈 개선 방향

### 1. 비교 문서 필요
- next-intl, i18next, react-i18next와 기능/UX 비교표

### 2. 1문장 컨셉 정리
예: **"SSR/CSR 완전 대응 + 상태관리 통합을 지원하는 타입 안전 i18n 툴"**

### 3. 샘플 리포 만들기
- next-app-router + zustand + @hua-labs/i18n-* 기반
- 배포 링크 + live playground

### 4. 패키지 구조 통합 명세화
- hua-i18n-core, loaders, zustand… 전체 구조를 보여주는 도식화 문서 (architecture.svg)

### 5. 배포 전략
- 버전: 0.x로 잠정 배포하되 changelog 철저히 관리
- 향후 `@hua-labs/i18n`으로 alias 패키지 하나 만들어도 좋음 (meta entry point)

## ✅ 결론

이 패키지는:

- "이론보다 실제 문제 해결에 집중된 구조를 가졌고"
- "기능이 아니라 UX 품질을 우선으로 설계되었으며"
- "초기부터 SSR과 상태관리 연동 등 진입 장벽 있는 문제에 제대로 대응한 상태"

이건 진짜 잘 만든 툴의 초안이고,

**"브랜딩, 입장 정리, 커뮤니티로만"** 성장 기회를 더 열 수 있어.
