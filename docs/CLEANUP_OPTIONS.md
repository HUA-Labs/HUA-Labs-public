# 코드베이스 정리 옵션 분석

### 빌드 성공 ✅

- `sum-chat` - 메인 프로덕션 앱

### 빌드 실패 ❌

- `hua-motion` - 개발/테스트 앱 (빌드 에러)

## 옵션 1: 지우기 (Cleanup)

### 삭제 대상 후보
1. **`hua-labs`** - 기본 Next.js 템플릿, 실제 사용 안 함
2. **`hua-demo`** - SUM API 데모, 필요시 재생성 가능
3. **`hua-ui`** - UI 패키지 사이트, Storybook으로 대체 가능

### 장점 ✅
- 코드베이스 정리 및 단순화
- 빌드 시간 단축
- 유지보수 부담 감소
- CI/CD 파이프라인 단순화
- 의존성 관리 용이

### 단점 ❌
- 나중에 필요할 수 있음
- 삭제 후 복구 어려움
- 데모/문서 목적 상실

### 추천 삭제 대상
- `hua-labs` - 기본 템플릿, 명확히 불필요
- `hua-demo` - 필요시 재생성 가능

### 보류 대상
- `hua-ui` - Storybook과 중복이지만 UI 패키지 문서화 목적
- `i18n-test`, `hua-motion` - 개발 중인 기능 테스트용

## 옵션 2: 빌드/린트 성공시키기 (Fix All)

### 수정 대상
1. **`sum-diary`** - `nickname_hash` 타입 에러 수정
2. **`i18n-test`** - Window 타입 확장
3. **`hua-motion`** - 빌드 에러 수정
4. **`hua-demo`** - 빌드 설정 수정
5. **`hua-labs`** - 빌드 설정 수정
6. **`hua-ui`** - 빌드 설정 수정

### 장점 ✅
- 전체 코드베이스 일관성 유지
- CI/CD 파이프라인 완전 통과
- 향후 개발 시 즉시 사용 가능

### 단점 ❌

- 일부 앱은 실제로 사용 안 할 수도 있음
