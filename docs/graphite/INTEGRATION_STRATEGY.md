# Cursor와 Graphite 통합 전략

## 개요

이 문서는 Cursor AI와 Graphite를 함께 사용하는 전략과 단계별 도입 계획을 설명합니다.

## 통합 배경

### Cursor 인수 현황

- **인수 발표**: 2025년 12월
- **인수 완료 예정**: 2025년 12월 말 ~ 2026년 초
- **완전 통합 예정**: 2026년 내 점진적 진행

### 현재 상태

- Cursor와 Graphite는 현재 독립적으로 운영됨
- 통합 기능은 아직 개발 중
- Hobby Plan으로 기본 기능 사용 가능

## AI 개발 사이클

### 완전한 AI 개발 사이클

```
Cursor AI (코드 작성)
    ↓
Graphite Agent (자동 리뷰)
    ↓
Stacked PRs (단계별 검증)
    ↓
자동 머지
```

### 각 단계의 역할

#### 1. Cursor AI - 코드 작성
- 코드 생성 및 수정
- 리팩토링 제안
- 버그 수정

#### 2. Graphite Agent - 코드 리뷰
- 자동 코드 리뷰
- 가이드라인 준수 확인
- 버그 및 이슈 발견

#### 3. Stacked PRs - 단계별 검증
- 작은 단위로 PR 분리
- 각 단계별 검증
- 독립적 머지

## Hobby Plan 제한사항 및 대응

### 제한사항

1. **AI 리뷰 제한**
   - Graphite Agent의 AI 리뷰가 제한적
   - 일부 고급 기능 미사용

2. **기능 제한**
   - Merge Queue 제한
   - 일부 자동화 기능 제한

### 대응 방안

1. **선택적 활용**
   - 중요한 PR에만 AI 리뷰 사용
   - 수동 리뷰와 병행

2. **커스텀 규칙 최적화**
   - 핵심 가이드라인만 규칙으로 설정
   - 효율적인 리뷰 포인트 선정

3. **단계적 업그레이드**
   - 필요 시 Starter Plan 고려 ($20/월)
   - 통합 완료 후 재평가

## 워크플로우 비교

### 현재 워크플로우 (Git만 사용)

```
1. Cursor AI로 코드 작성
2. Git 커밋
3. GitHub에 PR 생성
4. 수동 리뷰 (본인이 직접)
5. 머지
```

### Graphite 도입 후

```
1. Cursor AI로 코드 작성
2. Graphite로 커밋 및 PR 생성
3. Graphite Agent 자동 리뷰
4. 리뷰 피드백 반영 (Cursor AI 활용)
5. Stacked PRs로 단계별 머지
```

### 통합 완료 후 (예상)

```
1. Cursor AI로 코드 작성
2. Cursor 내에서 Graphite PR 생성
3. Graphite Agent 자동 리뷰 (Cursor 내 표시)
4. Cursor AI로 리뷰 피드백 즉시 반영
5. 원클릭 머지
```
