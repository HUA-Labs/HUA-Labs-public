# 디자인 결정 메모

## 구체적 결정 사항

이 문서는 설계 과정에서 내린 구체적인 결정사항을 기록합니다.

### 색상 결정

**Product Preset Primary 색상**
- **결정**: 밝은 파란색
- **이유**: 밝고 친근한 느낌, 신뢰감, 전문성
- **구현**: Tailwind의 `blue` 색상 팔레트 중 밝은 톤 사용

**Marketing Preset Primary 색상**
- **결정**: 보라색/핑크 계열
- **이유**: 화려함, 창의성, 눈에 띄는 느낌
- **구현**: Tailwind의 `purple` 또는 `pink` 색상 팔레트 사용

### 타이포그래피 결정

**H1 크기**
- **결정**: 2xl
- **이유**: 일단 이 크기로 진행, 향후 실제 사용하면서 조정 가능
- **구현**: Tailwind의 `text-2xl` 사용

**제목 계층 구조**
- H1: 2xl (메인 제목)
- H2: xl (섹션 제목)
- H3: lg (서브 섹션)
- H4: base (작은 제목)

### 간격 시스템 결정

**자동 설정 원칙**
- **결정**: spacing을 지정하지 않으면 Preset 기본값 자동 사용
- **이유**: 개발자가 매번 간격을 지정하지 않아도 되도록
- **구현**: 
  ```tsx
  // spacing이 없으면 Preset 기본값 사용
  <Stack>  // Product: md, Marketing: xl 자동 적용
  ```

**Preset별 기본 간격**
- Product: md (보수적, 컴팩트)
- Marketing: xl (넓은 간격, 여유로운 레이아웃)

## 향후 고려 사항

### 폰트 서브셋화
- 필요한 글자만 포함하여 폰트 파일 크기 최적화
- 예: 한글 기본 자모, 영문, 숫자만 포함
- 성능 최적화와 함께 디자인 품질 유지

### 색상 세부 조정
- 실제 사용하면서 밝은 파란색의 정확한 톤 조정 가능
- Tailwind 색상 팔레트 중 적절한 단계 선택 (blue-400? blue-500?)

### 타이포그래피 세부 조정
- H1 2xl이 실제로 적절한지 사용하면서 확인
- 필요시 크기 조정 가능
