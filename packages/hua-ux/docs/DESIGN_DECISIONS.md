# 디자인 결정 사항

## 개요

설계자(디자이너) 관점에서 hua-ux의 기본값을 정의하는 문서입니다. **"친절함을 목표로"** 개발자가 결정해야 하는 것들을 최소화하고, 좋은 기본값을 제공합니다.

## 기본 원칙

### 1. 친절함 우선
- 개발자가 매번 결정하지 않아도 되도록
- "그냥 쓰면 되는" 수준까지
- 하지만 필요하면 커스터마이징 가능

### 2. 디자이너가 정한 좋은 기본값
- 단순히 "기본값"이 아닌 "좋은 기본값"
- 각 Preset마다 일관된 디자인 언어
- 실제 프로덕션에서 사용 가능한 수준

### 3. 점진적 확장
- 처음엔 기본값만 제공
- 필요에 따라 커스터마이징 가능
- 향후 고급 기능 추가 (폰트 서브셋화 등)

## Preset별 디자인 방향

### Product Preset (제품 페이지용)

**컨셉**: 전문적, 신뢰감, 효율적

**색상**:
- Primary: **밝은 파란색** (신뢰감, 전문성, 밝고 친근한 느낌)
- Secondary: 회색 계열 (중립적)
- Accent: 보라색 계열 (차분한 강조)

**타이포그래피**:
- H1: **2xl** (메인 제목, 일단 이 크기로 진행)
- 제목: 적당한 크기, 명확한 계층
- 본문: 읽기 좋은 크기와 간격

**간격**:
- 보수적 (md 기본값)
- 컴팩트한 레이아웃

**모션**:
- 빠른 전환 (400ms 이하)
- 최소한의 딜레이
- 호버/클릭 인터랙션 최소화

### Marketing Preset (랜딩 페이지용)

**컨셉**: 화려함, 드라마틱, 눈에 띄는

**색상**:
- Primary: 보라색/핑크 계열 (화려함, 창의성)
- Secondary: 오렌지 계열 (에너지)
- Accent: 그라데이션 활용

**타이포그래피**:
- 제목: 큰 크기, 강한 임팩트
- 본문: 넓은 간격으로 읽기 편하게

**간격**:
- 넓은 간격 (xl 기본값)
- 여유로운 레이아웃

**모션**:
- 느린 전환 (800ms 이상)
- 긴 딜레이로 드라마틱한 효과
- 호버/클릭 인터랙션 강조

## 색상 시스템

### 기본 색상 팔레트

각 Preset마다 다음 색상들을 정의:

1. **Primary**: 주요 액션 색상 (버튼, 링크 등)
2. **Secondary**: 보조 색상 (배경, 보조 요소)
3. **Accent**: 강조 색상 (특별한 강조가 필요한 곳)
4. **Success**: 성공 상태 (완료, 성공 메시지)
5. **Warning**: 경고 상태 (주의, 경고 메시지)
6. **Error**: 에러 상태 (오류, 삭제 등)
7. **Info**: 정보 상태 (정보 메시지)

### 색상 적용 예시

```tsx
// Product Preset 사용 시
<Button>Click</Button>  // 자동으로 파란색 (primary)

// Marketing Preset 사용 시
<Button>Click</Button>  // 자동으로 보라색 (primary)
```

## 타이포그래피 시스템

### 계층 구조

1. **H1**: 메인 제목 (가장 큰 크기, 가장 굵게)
2. **H2**: 섹션 제목
3. **H3**: 서브 섹션 제목
4. **H4**: 작은 제목
5. **Body**: 본문 (가장 많이 사용)
6. **Small**: 작은 텍스트
7. **Caption**: 캡션, 설명 텍스트

### 향후 계획: 폰트 서브셋화

- 필요한 글자만 포함하여 폰트 파일 크기 최적화
- 예: 한글 기본 자모, 영문, 숫자만 포함
- 성능 최적화와 함께 디자인 품질 유지

## 간격 시스템

### 자동 설정 원칙

- 개발자가 간격을 지정하지 않으면 Preset 기본값 사용
- Product: 보수적 간격 (md)
- Marketing: 넓은 간격 (xl)

### 간격 적용 예시

**자동 설정 원칙**: spacing을 지정하지 않으면 Preset 기본값 자동 사용

```tsx
// spacing을 지정하지 않으면 Preset 기본값 사용
<Stack>  // Product: md 간격, Marketing: xl 간격 (자동 적용)
  <Card>Item 1</Card>
  <Card>Item 2</Card>
</Stack>

// 명시적으로 지정하면 그 값 사용 (Preset 무시)
<Stack spacing="lg">  // lg 간격 사용
  <Card>Item 1</Card>
  <Card>Item 2</Card>
</Stack>
```

## 컴포넌트 기본값

### Button
- **Product**: 파란색, 보통 크기, 적당한 둥글기
- **Marketing**: 보라색, 큰 크기, 둥근 모서리

### Card
- **Product**: 얇은 그림자, 적당한 여백
- **Marketing**: 두꺼운 그림자, 넓은 여백

### Badge
- **Product**: 작은 크기, 차분한 색상
- **Marketing**: 큰 크기, 화려한 색상

## 커스터마이징 가능성

### 기본값 우선, 커스터마이징 가능

```tsx
// 기본값 사용 (Preset 자동 적용)
<Button>Click</Button>

// 일부만 커스터마이징
<Button color="red">Delete</Button>  // 색상만 변경

// 완전 커스터마이징
<Button 
  color="red"
  size="lg"
  variant="outline"
>
  Custom Button
</Button>
```

## 구체적 결정 사항

### 색상
- **Product Preset Primary**: 밝은 파란색 (밝고 친근한 느낌, 신뢰감)
- **Marketing Preset Primary**: 보라색/핑크 계열 (화려함, 창의성)

### 타이포그래피
- **H1 크기**: 2xl (일단 이 크기로 진행, 향후 조정 가능)
- 제목 계층: H1(2xl) > H2(xl) > H3(lg) > H4(base)
- 본문: base 크기, 읽기 좋은 간격

### 간격
- **자동 설정**: spacing을 지정하지 않으면 Preset 기본값 자동 사용
- Product: md (보수적)
- Marketing: xl (넓은 간격)

## 구현 우선순위

### 1단계: 기본값 정의 (가장 중요)
- [ ] 색상 팔레트 정의
  - Product Primary: 밝은 파란색
  - Marketing Primary: 보라색/핑크 계열
- [ ] 타이포그래피 시스템 정의
  - H1: 2xl (일단 이 크기로 진행)
  - 제목/본문 계층 구조
- [ ] 간격 시스템 정의
  - 자동 설정: spacing 미지정 시 Preset 기본값 사용
- [ ] 컴포넌트 기본값 정의

### 2단계: 자동 적용
- [ ] PresetContext로 전역 적용
- [ ] 컴포넌트가 Preset 자동 감지 및 사용

### 3단계: 고급 기능 (향후)
- [ ] 폰트 서브셋화
- [ ] 커스텀 Preset 생성 기능
- [ ] 디자인 토큰 시각화 도구
