# 디버깅 실패 로그 작성 스킬

이 스킬은 디버깅 실패 로그를 작성하고 중요한 패턴을 별도 문서에 기록하는 방법을 안내합니다.

## 🚨 AI 어시스턴트 필수 준수 사항

### 디버깅 로그 작성 시 필수 확인

```
IF (디버깅 실패 시) THEN
  1. 실패한 방법과 이유 기록
  2. 시도한 해결 방법 순서대로 기록
  3. 최종 해결 방법 강조
  4. 패턴 추출 확인
END IF
```

### 패턴 문서화 제안

```
IF (같은 문제가 반복됨) THEN
  → "이 문제는 패턴 문서에 기록하는 것이 좋겠습니다. docs/patterns/ 폴더에 추가하시겠어요?"
END IF
```

## 디버깅 로그 작성 원칙

### 1. 실패 기록
- 디버깅 시도와 실패를 모두 기록
- 실패한 방법과 이유를 명확히 기록
- 시간과 노력을 투자한 내용 기록

### 2. 패턴 추출
- 반복되는 실패 패턴 식별
- 중요한 패턴은 별도 문서에 기록
- 향후 참고를 위한 구조화된 문서 작성

### 3. 해결 과정 기록
- 시도한 해결 방법 순서대로 기록
- 각 방법의 결과와 이유 기록
- 최종 해결 방법 강조

## 패턴 문서 작성

### 패턴 문서 위치

- **패턴 문서**: `docs/patterns/`
- **예시**: `docs/patterns/build-errors.md`, `docs/patterns/type-errors.md`

### 패턴 문서 구조

```markdown
# [패턴 이름] 패턴

**작성일**: YYYY-MM-DD  
**목적**: 반복되는 문제 패턴을 정리하여 개발 효율성 향상

---

## 문제 상황

[문제가 발생하는 상황 설명]

## 원인 분석

[문제의 근본 원인]

## 해결 방법

### 방법 1: [해결 방법]
[상세 설명]

### 방법 2: [대안 해결 방법]
[상세 설명]

## 예방 방법

1. 예방 방법 1
2. 예방 방법 2

## 관련 데브로그

- [데브로그 링크](./../devlogs/DEVLOG_YYYY-MM-DD_제목.md)

## 체크리스트

이 패턴을 만났을 때 확인할 사항:
- [ ] 체크 항목 1
- [ ] 체크 항목 2
```

## 디버깅 로그 작성 예시

### ✅ 올바른 예시: 빌드 오류

```markdown
# 디버깅 로그 - 2025-12-15 - Vercel 빌드 오류

## 문제 상황
Vercel에서 sum-diary 앱 빌드 시 `Module not found` 오류 발생

## 시도한 해결 방법

### 시도 1: 의존성 재설치
**결과**: 실패
**실패 이유**: 
- 로컬에서는 정상 작동
- Vercel 환경 차이

### 시도 2: turbo.json 설정 변경
**결과**: 실패
**실패 이유**: 
- 환경 변수 설정 문제
- 빌드 순서 문제

### 시도 3: 빌드 스크립트 수정
**결과**: 성공
**해결 방법**: 
- Prisma Client 생성 단계 추가
- 빌드 명령어를 `turbo run build`로 변경
```

## 패턴 문서 업데이트

### 패턴 문서에 추가

디버깅 로그에서 패턴을 발견했을 때:

1. `docs/patterns/` 폴더의 관련 문서 확인
2. 없으면 새 문서 생성
3. 패턴 추가:
   - 문제 상황
   - 원인 분석
   - 해결 방법
   - 예방 방법
   - 관련 데브로그 링크

### ✅ 올바른 예시: 패턴 문서

```markdown
# 빌드 오류 패턴

## 문제 상황
Vercel 빌드 시 모듈을 찾을 수 없는 오류

## 원인 분석
- Prisma Client가 빌드 전에 생성되지 않음
- 환경 변수가 빌드 타임에 접근 불가

## 해결 방법
1. `package.json`에 `prisma generate` 스크립트 추가
2. 빌드 전에 Prisma Client 생성 보장
3. 환경 변수를 Vercel에서 직접 관리

## 예방 방법
- 빌드 스크립트에 Prisma Client 생성 단계 포함
- 환경 변수는 Vercel/Doppler에서 관리
```
