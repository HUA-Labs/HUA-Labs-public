# Hue i18n Builder - 확장 기획

## 개요

Hue 에디터를 확장하여 **i18n 키 관리 + 다국어 JSON 자동 생성** 도구로 발전시키는 아이디어.

노코드로 UI 와이어프레임을 만들면서 동시에 i18n 키를 배정하고, AI가 키 이름과 네임스페이스를 추천해주는 시스템.

## 핵심 기능

### 1. 컴포넌트별 i18n 키 배정

```typescript
interface EditorNode {
  id: string;
  type: string;
  props?: Record<string, unknown>;
  children?: EditorNode[] | string;

  // i18n 확장
  i18n?: {
    key: string;           // e.g., "hero.title"
    namespace?: string;    // e.g., "landing"
    description?: string;  // 번역가를 위한 컨텍스트
  };
}
```

### 2. AI 연동 기능

- **키 이름 추천**: 컴포넌트 내용과 위치 기반으로 적절한 키 이름 제안
  - 예: "환영합니다" → `hero.welcome`, `home.greeting` 등 추천

- **네임스페이스 추천**: 페이지/섹션 구조 분석하여 네임스페이스 제안
  - 예: Header 내 → `common.nav.*`
  - 예: HeroSection 내 → `landing.hero.*`

- **유사 키 탐지**: 기존 번역 키와 유사한 텍스트가 있으면 재사용 제안
  - "로그인" 입력 시 → "이미 `auth.login` 키가 있어요. 재사용할까요?"

- **일괄 번역 요청**: 새로운 키들을 AI에게 번역 요청
  - 한국어 원본 → 영어, 일본어, 중국어 등 자동 생성

### 3. JSON 내보내기

```json
// ko.json
{
  "landing": {
    "hero": {
      "title": "환영합니다",
      "subtitle": "최고의 서비스를 경험하세요"
    },
    "features": {
      "title": "주요 기능"
    }
  },
  "common": {
    "nav": {
      "home": "홈",
      "about": "소개",
      "contact": "문의"
    }
  }
}

// en.json (AI 생성)
{
  "landing": {
    "hero": {
      "title": "Welcome",
      "subtitle": "Experience the best service"
    }
  }
}
```

## 워크플로우

```
1. 디자이너/기획자가 UI 와이어프레임 구성
              ↓
2. 각 텍스트 컴포넌트에 i18n 키 배정 (AI 추천)
              ↓
3. 유사 키 있으면 재사용 여부 결정
              ↓
4. 완성된 와이어프레임에서 JSON 내보내기
              ↓
5. AI가 다국어 번역 생성
              ↓
6. 번역가가 AI 번역 검수/수정
              ↓
7. 최종 i18n JSON 파일 생성 → 프로젝트에 적용
```

## 기술 스택

- **AI**: Claude API (키 추천, 유사도 분석, 번역)
- **유사도 검색**: 임베딩 기반 시맨틱 검색 또는 fuzzy matching
- **저장소**: 기존 i18n JSON 파일 또는 DB

## 장점

1. **개발자 부담 감소**: 키 이름 고민 없이 AI가 추천
2. **일관성 유지**: 유사 키 재사용으로 중복 방지
3. **협업 용이**: 기획자/디자이너도 i18n 작업 가능
4. **빠른 번역**: AI 초벌 번역 → 검수 워크플로우
